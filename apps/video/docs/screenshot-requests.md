# Launch-video screenshot requests — FINAL (union of the three storyboards)

All three boards render **100% recreated UI** — these captures are *fidelity references* for the Remotion builds (chrome geometry, engine-rendered text, organic content), not assets that appear in the film. That means complete/authentic **state** matters more than pixel-perfect staging.

**Tags**: `ALL` = needed whichever concept wins. `[B1]` = only if "Ships Its Own Issues" · `[B2]` = only if "Strike the Don't" · `[B3]` = only if "While You Were in Standup".

## Global capture rules

- **One shared fixture pass**: workspace `Exponential`, dev project `Exponential` (prefix `EXP`, indigo), user Dennis (DS) on the MacBook Pro. Board populated with all four status groups (In Progress / Todo / Backlog / Done), a mix of priorities (incl. one urgent), label chips, one due date, one unassigned row. Create **one release with a partial checklist** (e.g. 3 of 8 done) and put the hero issue in it. Exact titles/ids don't matter — the boards use their own fixture strings.
- **One live coding run feeds five shots**: start a real `Start coding` session on one issue and, while it runs, grab D2 (desktop dock), W1 (web steer), M1 (iOS steer), plus the board row's `coding` pill; then let it finish for the exit-0 frame. Don't tear it down between captures.
- Desktop/web: capture at **2× DPR**, window ≈ **1440×900–1600×1000 logical** (matches the existing `apps/video/public/shots/*` at 2880×1800; the boards stage the desktop at 1536–1600px design width). PNG, full window.
- iOS: iPhone Pro simulator or device, portrait, native @3×. Android: not needed by any board.
- English locale, dark theme (forced anyway).
- Existing `apps/video/public/shots/01-login…05-inbox` stay valid; nothing below duplicates them. The earlier draft's W2 (rich-markdown issue detail), W3 (web sidebar badges), M3 (Android hero), and the desktop Source-Control commit/conflict shots are **dropped — no board stages them**.
